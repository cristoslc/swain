# swain-status roadmap integration

## Problem Statement

swain-status currently aggregates specgraph, tk, git, and GitHub issues but has no awareness of the Eisenhower-prioritized roadmap. The operator must separately run `chart.sh roadmap` and read ROADMAP.md to see priority decisions. The status output should surface the top decision items from the roadmap contextually — filtered by focus lane when set, showing "needs activation", "needs decomposition", or "ready to complete" signals inline.

## External Behavior

**Inputs:** Existing swain-status context (specgraph, tk, git, GitHub issues), ROADMAP.md (generated by `chart.sh roadmap`), focus lane setting (if set).

**Output:** swain-status output augmented with a "Roadmap Decisions" section showing top decision items contextually — filtered by focus lane when set, otherwise top 3-5 items from Do First and Schedule quadrants.

**Preconditions:** `chart.sh roadmap` is available and produces a valid ROADMAP.md.

**Postconditions:** ROADMAP.md is refreshed (if stale) and decision items are surfaced as actionable prompts in the status output.

**Constraints:**
- Roadmap refresh is skipped if ROADMAP.md is newer than all `docs/**/*.md` files (same staleness check as specgraph cache)
- Decision items are surfaced as prompts, not just listed

## Acceptance Criteria

- **Given** ROADMAP.md is stale or missing, **When** swain-status runs, **Then** it invokes `chart.sh roadmap` to refresh before rendering
- **Given** a focus lane is set, **When** swain-status renders roadmap decisions, **Then** only items under that Vision appear
- **Given** no focus lane, **When** swain-status renders, **Then** top 3-5 highest-priority decision items from Do First and Schedule quadrants are shown
- **Given** items with "needs decomposition" or "activate or drop" decisions, **When** swain-status renders, **Then** these are surfaced as actionable prompts, not just listed
- The roadmap refresh is skipped if ROADMAP.md is newer than all `docs/**/*.md` files (same staleness check as specgraph cache)
