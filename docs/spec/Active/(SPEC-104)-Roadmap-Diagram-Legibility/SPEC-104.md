# Roadmap diagram legibility

## Problem Statement

The current Mermaid diagrams in ROADMAP.md have legibility issues: quadrant chart labels overlap when items cluster at similar coordinates, Gantt task labels truncate at narrow widths, and the dependency graph's color-coded nodes wash out in dark-mode editors (Typora, VS Code dark themes). The roadmap must be readable without squinting or switching themes.

## External Behavior

**Inputs:** ROADMAP.md as generated by `chart.sh roadmap` (SPEC-102), Mermaid rendering contexts (Typora, GitHub, VS Code dark/light themes).

**Output:** ROADMAP.md with diagrams that meet legibility standards across rendering contexts without requiring theme changes or zoom adjustments.

**Preconditions:** `chart.sh roadmap` produces Mermaid diagrams.

**Postconditions:** Diagrams pass contrast and label overlap checks at standard rendering widths.

**Constraints:**
- Must remain valid Mermaid syntax
- Must not require external tooling beyond what `chart.sh` already uses
- Label shortening must preserve artifact IDs (e.g., EPIC-038 must remain identifiable)

## Acceptance Criteria

- **Given** a quadrant chart with 20+ items, **When** rendered in Typora or GitHub, **Then** no more than 2 labels overlap at default zoom
- **Given** a Gantt chart, **When** rendered at 800px width, **Then** all task labels are fully readable (no mid-word truncation)
- **Given** the dependency graph, **When** rendered in a dark-mode editor, **Then** node colors maintain sufficient contrast (WCAG AA minimum 4.5:1)
- **Given** `chart.sh roadmap`, **When** it detects the quadrant chart would have overlapping labels, **Then** it applies label shortening or positional jitter to reduce collisions

## Scope & Constraints

Investigate Mermaid theming/config options, label shortening strategies (abbreviations, ID-only with legend), positional jitter algorithms, and whether ROADMAP.md should optionally embed pre-rendered PNGs alongside or instead of inline Mermaid blocks for offline/dark-mode compatibility.

Does NOT change the data model or command interface of `chart.sh roadmap` — that is governed by SPEC-102.
