# Agent Client Protocol Python SDK — A New Chapter

## Overview

A blog post by Chojan Shang (PsiACE), the maintainer of the ACP Python SDK, announcing its official adoption into the `agentclientprotocol` GitHub organization. Published November 7, 2025.

- **URL:** https://psiace.me/posts/acp-python-sdk-new-chapter/
- **Author:** Chojan Shang (PsiACE)
- **Date:** November 7, 2025

## History

The Python SDK began as a curiosity-driven side project to understand how coding agents communicate. Shang developed it while exploring "how state, intent, and messages flow between the client and the agent." It gradually matured into a production-ready tool adopted by organizations like Moonshot.

## Official adoption

As of November 2025, the Python SDK officially moved under the `agentclientprotocol` organization, joining the TypeScript, Rust, and Kotlin SDKs as part of the official ecosystem. This transition signals that the Python SDK is no longer a community experiment but a first-class, officially maintained implementation.

## Maturity signals

- The SDK tracks the evolving ACP specification and serves as "a foundation for real-world integrations."
- Production adopters (Moonshot) use it in shipping products.
- The maintainer's stated priorities are "reliability, clarity, and making it easier for developers to build agentic systems."
- Migration guides exist for version upgrades (0.7 and 0.8 migrations documented).

## Relevance to swain

This source confirms the Python SDK's legitimacy and maturity. It is not a wrapper or shim; it is an official, production-tested SDK with a dedicated maintainer and a clear upgrade path. The November 2025 adoption into the official org predates the current project timeline, meaning the Python SDK has had months of stabilization as an official artifact.
