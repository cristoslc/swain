# Coolify -- Self-Hosted PaaS Overview

Coolify is an open-source, self-hosted Platform-as-a-Service (PaaS) alternative to Vercel, Heroku, and Netlify. It wraps Docker, a reverse proxy, SSL certificate management, and deployment pipelines into a single web dashboard.

## Architecture

- **Reverse proxy**: Traefik (default) or Caddy, configured automatically through the dashboard. No manual Nginx configs needed.
- **TLS**: Automatic Let's Encrypt certificate issuance and renewal for custom domains. Wildcard SSL supported via DNS challenge.
- **DNS**: Manual A record setup required. Coolify does not manage DNS records directly, but supports wildcard domains and autogenerated subdomains when wildcard DNS is configured.
- **Tunnel**: Not built in. Coolify has a documented integration with Cloudflare Tunnels to expose resources without public IPs or port forwarding. Supports both single-resource and all-resource tunnel modes.
- **Container orchestration**: Docker and Docker Swarm. Supports Docker Compose for multi-service stacks.

## Ingress Chain Coverage

| Layer | Covered? | How |
|-------|----------|-----|
| DNS | Partial | Requires manual A record; autogenerated subdomains with wildcard DNS. |
| TLS | Yes | Automatic Let's Encrypt via Traefik or Caddy. |
| Reverse proxy | Yes | Traefik or Caddy, host-header-based routing. |
| Tunnel | No (integration) | Documented Cloudflare Tunnel integration, not native. |
| Container | Yes | Docker / Docker Compose / Swarm. |

## Composability

Coolify is a **monolithic platform**, not a library or composable toolkit. You use it as a whole or not at all. The dashboard is the primary interface. An API exists but is incomplete as of 2026. No CLI-first workflow.

## Solo Operator Burden

- Installation: one-liner Docker script.
- Resource overhead: ~500-700 MB RAM idle, 5-6% CPU. Heavier than Dokku or Dokploy.
- Maintenance: automatic updates available. Web dashboard for all operations.
- 35,000+ GitHub stars, active development.

## Comparison to Commodore

- **Scope overlap**: Coolify covers proxy + TLS + container but not DNS provisioning or tunnel creation. Commodore covers DNS + proxy + TLS + container with a tunnel gap.
- **Architecture**: Coolify is a monolithic platform with a web UI. Commodore uses hexagonal architecture with swappable adapters.
- **Composability**: Coolify cannot be used as a library. Commodore's port/adapter model allows individual components to be swapped or extended.
- **DNS automation**: Coolify requires manual DNS. Commodore automates DNS via CloudflareDNS adapter.

## Key Takeaway

Coolify is the most feature-rich open-source self-hosted PaaS, but it trades composability for convenience. It covers 3 of 4 ingress layers (proxy, TLS, container) and integrates with Cloudflare Tunnels for the fourth. Not usable as a library or component.
