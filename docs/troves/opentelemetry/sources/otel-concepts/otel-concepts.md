# OpenTelemetry Concepts

Key concepts in OpenTelemetry.

This section covers data sources and key components of the OpenTelemetry
project, which can help you understand how OpenTelemetry works.

## Sub-topics

- **[Observability primer](https://opentelemetry.io/docs/concepts/observability-primer/)** -- Core observability concepts.
- **[Context propagation](https://opentelemetry.io/docs/concepts/context-propagation/)** -- Learn about the concept that enables Distributed Tracing.
- **[Signals](https://opentelemetry.io/docs/concepts/signals/)** -- Learn about the categories of telemetry supported by OpenTelemetry.
- **[Instrumentation](https://opentelemetry.io/docs/concepts/instrumentation/)** -- How OpenTelemetry facilitates instrumentation.
- **[Components](https://opentelemetry.io/docs/concepts/components/)** -- The main components that make up OpenTelemetry.
- **[Semantic Conventions](https://opentelemetry.io/docs/concepts/semantic-conventions/)** -- Common names for different kinds of operations and data.
- **[Resources](https://opentelemetry.io/docs/concepts/resources/)**
- **[Instrumentation scope](https://opentelemetry.io/docs/concepts/instrumentation-scope/)**
- **[Sampling](https://opentelemetry.io/docs/concepts/sampling/)** -- Learn about sampling and the different sampling options available in OpenTelemetry.
- **[Distributions](https://opentelemetry.io/docs/concepts/distributions/)** -- A distribution, not to be confused with a fork, is a customized version of an OpenTelemetry component.
- **[Glossary](https://opentelemetry.io/docs/concepts/glossary/)** -- Definitions and conventions for telemetry terms as used in OpenTelemetry.
