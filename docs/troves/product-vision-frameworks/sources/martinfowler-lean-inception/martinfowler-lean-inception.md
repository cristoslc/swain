# Writing Product Vision in Lean Inception

## Geoffrey Moore Template (from Crossing the Chasm)
"For [final client], whose [problem that needs to be solved], the [name of the product] is a [product category] that [key-benefits, reason to buy it]. Different from [competition alternative], our product [key-difference]."

## Facilitation Process
1. Display the template visibly
2. Organize participants into small groups, each filling different sections
3. Synthesize contributions into a unified statement
4. Full team refines language collaboratively

## Key Note
Initial results typically lack logical flow. Refinement is essential — teams should freely modify all contributions to create a coherent final statement.

## The Elevator Test
A good product vision passes the "elevator test" — you can explain the project to someone within two minutes. If you can't, the vision isn't clear enough.
