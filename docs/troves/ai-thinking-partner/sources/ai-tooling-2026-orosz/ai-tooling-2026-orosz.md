# AI Tooling for Software Engineers in 2026

**Authors:** Gergely Orosz and Elin Nilsson
**Publication:** The Pragmatic Engineer (Substack)
**Date:** March 3, 2026
**Reading time:** ~25 minutes (free portion)
**Survey size:** ~900 respondents (January 27 - February 17, 2026)

## Overview

A comprehensive survey of nearly 1,000 software engineers on their actual AI tool usage, preferences, and experiences. The data provides one of the most detailed snapshots of real-world AI adoption in software engineering as of early 2026.

## Top 10 Findings

1. **Claude Code went from zero to #1 in eight months.** Released May 2025, it's already the most-used AI coding tool, overtaking GitHub Copilot and Cursor.
2. **AI is now mainstream.** 95% of respondents use AI tools at least weekly; 75% use AI for at least half their work.
3. **Cursor is catching up on GitHub Copilot.** 35% growth in mentions since the previous survey nine months ago.
4. **Most engineers juggle multiple tools.** 70% use 2-4 tools simultaneously; 15% use 5+.
5. **Staff+ engineers are the heaviest agent users.** 63.5% use agents regularly (vs 49.7% for regular engineers).
6. **Codex has explosive early growth.** Despite not existing during the last survey, OpenAI's Codex already has 60% of Cursor's usage.
7. **Using agents correlates with positive AI sentiment.** Agent users are nearly 2x as likely to feel excited; non-users are 2x as likely to be skeptical.
8. **Company size influences tool choice.** 10K+ companies default to Copilot (56%); tiny startups favor Claude Code (75%).
9. **Tight chatbot race.** ChatGPT, Gemini, and Claude have nearly equal standalone chatbot mentions.
10. **Directors and senior leaders especially favor Claude Code.** Twice as popular with Director+ as with less senior levels.

## Most-Used Tools (ranked by mentions)

1. **Claude Code** -- terminal-first coding agent from Anthropic
2. **Chatbots** (ChatGPT, Claude, Gemini, others)
3. **GitHub Copilot** -- the "oldest" AI coding tool (launched 2021)
4. **Cursor** -- agent-powered coding IDE
5. **Codex** -- OpenAI's coding agent
6. **Gemini CLI** -- Google's command line agent
7. **OpenCode** -- most popular open source coding agent
8. **Antigravity** -- Google's agentic IDE (from Windsurf team)
9. **JetBrains Junie**
10. **Zed** -- fast editor with agentic workflows

### 9-Month Trend

Claude Code has massive momentum -- jumped from nowhere to #1. Cursor grew ~35%. GitHub Copilot usage stable. Codex has 60% of Cursor's usage despite being brand new. OpenCode, Gemini CLI, and Antigravity are up-and-coming (~10% each).

## Popular Models

**Anthropic dominates coding tasks.** Opus 4.5 and Sonnet 4.5 (latest at survey time) come up more often than all other models combined. This is not even a contest.

~1 in 8 respondents just use whatever model is default at their company. Other mentions: Cursor's custom models, Kimi/Moonshot, DeepSeek variants, Qwen, Grok, Mistral.

## AI Adoption: Mainstream Achieved

- **95%** use AI tools weekly or more
- **75%** use AI for at least half their work
- **56%** do 70%+ of their work using AI
- Only **2.1%** don't use AI at all

## AI Agent Usage Rising

**55% regularly use AI agents** -- massive jump from 18 months ago when it was "one or two respondents."

Common agent use cases:
- Code review and validation
- Automating manual/annoying tasks
- Bug fixing and investigation
- Code investigation
- Debugging
- Code "crafting" and "weaving"

Typical setup: split-screen with Claude Code terminal + IDE for reviewing changes.

### Agent Usage by Level

| Level | Regular Agent Usage |
|-------|-------------------|
| Engineers | 49.7% |
| Leads/EMs | 46.1% |
| Directors/VPs | 51.9% |
| Staff+ | 63.5% |

### Agent Usage Correlates with AI Sentiment

- Agent users: 61% excited about AI, 11% skeptical
- Non-agent users: 36% excited, 22% skeptical

## Company Size and Tool Usage

- **Tiny companies (1-10)**: Claude Code 75%, Chatbots 55%, Cursor 42%, Copilot 35%
- **Large companies (10K+)**: Copilot overtakes Claude Code -- Microsoft enterprise sales and bundling
- **Google tools**: Gemini CLI and Antigravity stable at ~10% across all company sizes
- Cursor and OpenCode drop at massive companies (may build internal agents: Goose at Block, Meta's agent, Google's Jetski/Cider)

## Tools Engineers Love

Top "most loved" tools:
1. **Claude Code**: 46%
2. **Cursor**: 19%
3. **GitHub Copilot**: 9%
4. Claude models (standalone): 11%

Claude Code is especially loved by Director+ folks. Cursor gets less love as seniority increases. OpenCode rivals GitHub Copilot in "loved" mentions among Staff+ engineers.

## Demographics

- 55% engineers, 34% engineering leadership
- Median experience: 11-15 years
- Fairly even split across company sizes
- Most respondents in Europe or US

## Relevance to AI as Thinking Partner

This survey provides essential calibration data:

1. **AI adoption is no longer debatable.** 95% weekly usage among professional engineers.
2. **Experience correlates with agent usage.** Staff+ engineers -- the most experienced -- are the heaviest agent users, contradicting narratives that AI is mainly for juniors.
3. **Tool choice is not purely technical.** Enterprise procurement drives Copilot; individual preference drives Claude Code. The tool you use may reflect your company's purchasing process, not the tool's capability.
4. **The agent inflection point.** 55% regular agent usage is a tipping point -- the majority of professional developers now regularly delegate to AI agents.
5. **Sentiment-usage feedback loop.** Using agents makes you more positive about AI; not using them makes you more skeptical. This creates divergent camps with limited common ground.
