# Per-Vision and Per-Initiative Roadmap Slices

**SPEC:** SPEC-143
**Date:** 2026-03-21
**Status:** Approved

## Summary

Add `chart.sh roadmap --scope <ARTIFACT-ID>` to generate per-Vision and per-Initiative roadmap slices written to each artifact's folder. Replace the existing `--focus` flag. On project-wide roadmap refresh, regenerate all slices automatically.

## Decisions

1. **Replace `--focus` with `--scope`** — `--focus` filtered the project-wide ROADMAP.md to one Vision. `--scope` subsumes this and adds per-artifact slice output. No deprecation shim; `--focus` has no external consumers.
2. **Intent summary cascade** — script checks `brief-description` frontmatter (SPEC-144) first. If absent, writes `{{INTENT: <ARTIFACT-ID>}}` placeholder. swain-roadmap skill post-processes placeholders by reading source artifacts.
3. **Always regenerate all slices** — `chart.sh roadmap` (no flags) writes ROADMAP.md and all per-artifact slices. No opt-out flag; artifact count is small and file writes are cheap.

## Architecture

### Data flow

```
chart_cli.py: --scope replaces --focus
    |
roadmap.py: collect_roadmap_items(scope=ID) filters to subtree
    |
render_scoped_roadmap() -- new function, new Jinja template
    |
Writes roadmap.md to artifact folder
```

When no `--scope`: after writing ROADMAP.md, iterate all Vision and Initiative nodes, call the scoped renderer for each.

### Output locations

- Project-wide: `ROADMAP.md` (project root, unchanged)
- Per-artifact: `roadmap.md` in the artifact's folder (e.g., `docs/vision/Active/(VISION-001)-Foo/roadmap.md`)

### Scoped slice format

New Jinja template: `roadmap-slice.md.j2`

```markdown
<!-- Auto-generated by chart.sh roadmap --scope. Do not edit. -->
# VISION-001: Personal Agent Platform

> {brief-description or {{INTENT: VISION-001}} placeholder}

## Children

| Artifact | Title | Phase | Progress |
|----------|-------|-------|----------|
| [INITIATIVE-005](relative-link) | Operator Situational Awareness | Active | 3/7 |

## Progress

<progress-bar> 3/7 complete (43%)

## Recent Activity

- `abc1234` docs(spec): SPEC-143 roadmap slices
- `def5678` fix(roadmap): legend epic names
- `ghi9012` feat(roadmap): decision sections

## Priority Subset

{Eisenhower table rows filtered to this artifact's children}
```

### Intent summary resolution

Priority cascade in `roadmap.py`:
1. Check node for `brief-description` frontmatter field -- use if non-empty
2. Otherwise -- write `{{INTENT: <ARTIFACT-ID>}}`

`swain-roadmap` skill post-processes: scan for `{{INTENT: ...}}` markers, read each source artifact, extract first sentence of Value Proposition (Visions) or Goal/Objective (Initiatives), replace the marker.

### Backup of existing manual roadmaps

On first write to an artifact folder, if `roadmap.md` already exists and does NOT contain the `<!-- Auto-generated by chart.sh roadmap --scope. Do not edit. -->` header, copy to `roadmap.manual-backup.md` before overwriting.

### swain-roadmap skill changes

- Accept optional argument: `swain-roadmap VISION-001` passes `--scope VISION-001` to chart.sh
- After chart.sh completes, scan output for `{{INTENT: ...}}` markers and replace them
- Open the scoped file (not ROADMAP.md) when `--scope` is provided

### Vision definition harmonization

Update `vision-definition.md`: change "Every Vision SHOULD include ... a `roadmap.md`" to say the `roadmap.md` is auto-generated by `chart.sh roadmap --scope` and should not be manually maintained.
